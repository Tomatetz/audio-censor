# Stream Censor

Локальный прототип для стримеров: задерживает голос, распознаёт русскую речь и
проигрывает запрещённые слова задом наперёд до передачи аудио в OBS.

## Как это работает

1. Приложение получает звук с микрофона.
2. Оригинальный звук хранится в буфере (по умолчанию 5 секунд).
3. `faster-whisper` распознаёт свежие фрагменты и возвращает временные метки слов.
4. Совпадения со словарём помечаются на общей временной шкале.
5. При воспроизведении задержанного звука отмеченные слова разворачиваются
   задом наперёд. Также доступны режимы тишины и сигнала.

Видео следует задержать в OBS на то же число миллисекунд.

## Установка

Нужен Python 3.9+.

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

Для передачи обработанного звука в OBS на macOS удобно использовать виртуальное
аудиоустройство [BlackHole 2ch](https://github.com/ExistentialAudio/BlackHole).

## Запуск

### Графический интерфейс

```bash
python web_gui.py
```

После установки зависимостей можно дважды нажать `start_gui.command` в Finder.
Откроется локальная
страница в браузере; интернет для неё не нужен. В интерфейсе доступны
выбор устройств, основные настройки, запуск и остановка, журнал, тестовый текст
и папка записей. Кнопка **Закрыть приложение** останавливает фильтр и локальный
сервер, после чего закрывает текущую вкладку браузера и окно Terminal, из
которого был запущен `start_gui.command`. Другие окна Terminal не затрагиваются.

### Консоль

Сначала посмотреть номера аудиоустройств:

```bash
python app.py --list-devices
```

Запишите номера устройств и остальные параметры в `config.jsonc`, затем
запускайте фильтр без длинной команды:

```bash
python app.py
```

Пример основных настроек:

```jsonc
{
  // Микрофон и устройство вывода
  "input_device": 0,
  "output_device": 1,
  "delay": 5.0,
  "model": "small",
  "beam_size": 5,
  "debug_transcript": true
}
```

Параметры командной строки по-прежнему работают и временно переопределяют
значения из файла. Другой файл можно выбрать через `--config путь.json`.

Чтобы не воспроизводить звук и только записывать обработанный WAV:

```jsonc
"output_device": null
```

- `--delay` — задержка звука в секундах.
- `--mode` — способ замены: `reverse`, `beep`, `bark`, `meow` или `mute`.
- `--model` — модель Whisper (`tiny`, `base`, `small`, `medium`).
- `--compute-type` — обычно `int8`; на совместимом GPU можно выбрать другой.
- `--chunk` — размер окна распознавания.
- `--scan-every` — период запуска распознавания.
- `--sample-rate` — частота аудиоустройства, по умолчанию `48000`.
- `--beam-size` — число вариантов распознавания; `3` по умолчанию, `5` точнее,
  но тяжелее для процессора.
- `--debug-transcript` — выводить в терминал сырой распознанный текст.
- `--debug-words` — выводить отдельные word-токены и отметку `MATCH`.
- `--safety-margin` — желательный запас до выхода найденного слова; при
  недостаточном запасе приложение выводит `[risk]`.
- `--record-output` / `--no-record-output` — включить или отключить запись
  обработанного звука.
- `--record-transcript` / `--no-record-transcript` — сохранять TXT-журнал с
  временными метками ASR и реально заменённых слов.
- `--recordings-directory` — папка для WAV-записей.

При обычном запуске обработанный звук автоматически сохраняется в папку
`recordings`. Рядом создаётся одноимённый TXT-файл с метками `[ASR]`,
`[CENSOR:режим]`, `[MISS]`, `[RISK]` и `[LATE]`.

Для режимов `bark` и `meow` используются по четыре настоящих звуковых сэмпла.
Вариант выбирается случайно для каждого найденного слова. Источники и лицензии
перечислены в `assets/sounds/ATTRIBUTION.md`.

В OBS добавьте выходное виртуальное устройство как Audio Input Capture. Для
видео используйте фильтр **Video Delay (Async)** со значением `5000 ms`
(обычный **Render Delay** рассчитан только на небольшие задержки). Если фильтра
нет у сцены, поместите источники видео во вложенную сцену и примените задержку к
ней. Если распознавание иногда опаздывает, увеличьте обе задержки до 7–8 секунд.

Строка `[late]` в терминале означает, что Whisper завершил распознавание уже
после выхода слова из буфера. Для окна `--chunk 3` практический минимум —
`--delay 5`.

## Словарь

В `words.txt` одна запись на строку. Поддерживаются:

- обычные слова: `слово`;
- префиксы для словоформ: `корень*`;
- регулярные выражения: `re:шаблон`.

Начальный словарь намеренно небольшой. Его нужно настроить под речь конкретного
стримера, иначе неизбежны ложные срабатывания.

Обычные слова и префиксы из словаря передаются через специальный параметр
`hotwords`. Они не добавляются в текстовую подсказку модели, чтобы Whisper не
начинал перечислять содержимое словаря вместо реально произнесённой речи.

## Ограничения MVP

- Обрабатывается один микрофонный канал.
- Первая загрузка модели требует интернет.
- Замена голосом пока не реализована.
- OBS синхронизируется вручную одинаковой задержкой аудио и видео.
- Результат зависит от мощности компьютера, модели и манеры речи.
